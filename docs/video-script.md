# 3-minute pitch script

Hi, I built MaintShield, a confidential reward layer for maintenance teams using the Zama Protocol.

In real factories, maintenance tasks can be visible: which machine failed, who was assigned, and whether the work was approved. But rewards, KPI scores, and personal performance should not be visible to everyone.

MaintShield separates public workflow from private rewards.

A manager can add manager wallets and whitelist technician wallets. The manager creates a task and can assign it only to a whitelisted technician. The technician submits the completed work. The manager verifies the real-world repair and approves it. Only after approval, the manager assigns an encrypted reward value.

The important part is privacy: the task status remains public, but the reward amount is encrypted with Zama. The assigned technician can decrypt their own reward, while other wallets cannot see it.

This can be extended to maintenance bonuses, confidential KPI scoring, private payroll, contractor payments, and operations incentives.

MaintShield shows a real-world use case: public maintenance accountability, private personal rewards.
